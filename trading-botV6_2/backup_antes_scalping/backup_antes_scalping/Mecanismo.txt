4. Detección de close/open (doble validación)
Primaria — Candle freshness tracker
Cada vez que _on_new_candle() se dispara (1m), se actualiza un diccionario:
last_candle: dict[symbol, datetime] = {}

    if tf == "1min":
        self.last_candle[symbol] = candle_time
Secundaria — Spread anomaly
Si el spread se dispara > 5x el spread normal del símbolo, es señal de cierre inminente. Útil para BLACKOUT_PRE.
spike_ratio = current_spread / avg_spread
if spike_ratio > 5.0 and state == OPEN:
    transition_to(BLACKOUT_PRE)
Terciaria — Caída libre de volumen
Si el tick volume del último candle 1m es < 10% del promedio, mercado zombie. Transición a BLACKOUT_PRE también.
5. Lógica de transiciones (pseudocódigo)
def _update_state(self, sym: str, now: datetime):
    asset_class = self.asset_classes[sym]
    if asset_class == 'crypto':
        return  # siempre OPEN, nada que hacer

    last = self.last_candle.get(sym)
    if last is None:
        return  # aun no hay datos, esperar

    age_min = (now - last).total_seconds() / 60.0
    current_state = self.states[sym]

    # --- DETECCION DE APERTURA (CLOSED -> COOLDOWN) ---
    if current_state in (CLOSED, BLACKOUT_PRE) and age_min < 2:
        # Llego un candle fresco -> el mercado acaba de abrir
        self.states[sym] = COOLDOWN_POST
        self.cooldown_end[sym] = now + timedelta(minutes=10)
        return

    # --- DETECCION DE CIERRE INMINENTE (OPEN -> BLACKOUT_PRE) ---
    if current_state == OPEN and age_min > 25:
        # 25+ min sin candle fresco -> probablemente cerro
        self.states[sym] = BLACKOUT_PRE
        self.blackout_end[sym] = now  # se usara solo si hay reapertura
        return

    # --- COOLDOWN_POST -> OPEN (pasaron 10 min) ---
    if current_state == COOLDOWN_POST and now >= self.cooldown_end.get(sym, now):
        self.states[sym] = OPEN
        return

    # --- OPEN -> CLOSED (30+ min sin candle) ---
    if current_state in (OPEN, COOLDOWN_POST) and age_min > 30:
        self.states[sym] = CLOSED
        return

    # --- CLOSED -> esperando candle fresco ---
    # (no se hace nada, el primer candle fresco dispara COOLDOWN)
6. Pregunta clave que decide todo
def can_open_new_position(self, sym: str) -> bool:
    """¿Puedo colocar una nueva orden NOW para este símbolo?"""
    state = self.states.get(sym, OPEN)
    return state == OPEN  # los demas estados: NO

def should_evaluate(self, sym: str) -> bool:
    """¿Vale la pena correr el pipeline completo?"""
    state = self.states.get(sym, OPEN)
    if self.asset_classes.get(sym) == 'crypto':
        return True
    # Tradicional: evaluar solo si hay posibilidad de entrar o gestionar
    return state in (OPEN, COOLDOWN_POST, BLACKOUT_PRE)
can_open_new_position → se consulta en el momento de colocar la orden (último paso del pipeline), NO al inicio. Así el pipeline completo se ejecuta, se genera señal, se calcula volumen, y justo antes de enviar la orden se pregunta si está permitido. Esto permite que señales de salida (TP/SL dinámico, trailing) sigan funcionando aunque el mercado esté en BLACKOUT_PRE.

11. Warming — evitar falsos BLACKOUT_PRE
Los primeros 60 minutos después del arranque del bot, el detector de freshness no transiciona a CLOSED/BLACKOUT_PRE porque no tiene baseline de cuándo fue el último candle real. Usa un estado BOOTSTRAP donde solo recolecta datos pero nunca bloquea.

Sí, absolutamente. Ese es precisamente el punto fuerte del enfoque.
Para el viernes (cierre):
- Último candle 1m se genera el viernes ~15:59 (según huso del broker).
- A las 16:01 ya tiene 2 min de edad. A las 16:30 tiene 30+ min.
- El freshness tracker transiciona automáticamente a CLOSED sin saber qué día es.
- Dentro de esos 30 min (entre los 25 y 30 min sin candle), pasa por BLACKOUT_PRE.
Para el domingo (apertura):
- A las 16:00:01 llega el primer candle 1m fresco después de ~48h sin ninguno.
- age_min pasa de 2880 min a < 1 min → detecta el flanco de subida.
- Transiciona a COOLDOWN_POST automáticamente.
- 10 min después → OPEN.
No hay lógica de días de la semana, ni de horas fijas, ni de timezones. El sistema no sabe qué es viernes ni domingo. Solo ve "no llegaron candles por 48h → CLOSED" y luego "llegó un candle fresco → asumo que abrió". Funciona para:
- Fines de semana (forex cierra viernes, abre domingo)
- Feriados locales (ej. mercado de EU cerrado un lunes por Presidents' Day)
- Cierre intradiario (ej. algunos índices tienen sesión de 9:30 a 16:00 ET, el resto del día no hay candles)
- Problemas de conectividad con el broker (si MT5 deja de recibir datos, el bot también se pausa solo)
- Símbolos eliminados o renombrados (nunca más llegan candles → se quedan en CLOSED permanentemente sin errores)
Un solo mecanismo, cero supuestos, cero configuración.
